## 슬라이드 2


| 개요 | 서비스 품질의 표준화, 운영 효율성 향상, 비용 절감을 목적으로 설계된 PDCA(Plan-Do-Check-Act) 기반  서비스 운영 시스템의 제품 요구사항을 정의합니다. 본 시스템은 현장 데이터, 디지털 체크리스트 및 AI 기반 분석을 활용하여  서비스 운영의 디지털 전환을 실현합니다. |
| --- | --- |
| 문제 | 서비스 품질관리의 구조적 한계 서비스 품질관리 표준화 부재 인적 서비스 불확실성 확대 → 리스크관리 비용 상승 직원 부족 + 외국인 근로자 증가 법·안전·위생 규제는 강화 근본적 원인은 서비스 운영이 사람의 경험과 감(암묵지)에 의존하고 있기 때문 |
| 목표 | 품질 향상: 표준화된 프로세스와 지속적인 개선 활동을 통한 서비스 품질 강화 운영 효율성 증대: 업무 디지털화 및 보고 자동화를 통한 효율성 제고 비용 절감: 자원 할당 최적화 및 인적 오류 최소화를 통한 운영 비용 절감 데이터 기반 의사결정: 포괄적인 데이터 수집 및 분석을 통한 전략적 의사결정 지원 |
| 주요사용자 | 현장 관리자 (Field Managers) CS 담당자 (CS Representatives) 서비스 실무진 (Service Staff) 경영진 (Management) |
| 주요기능 | 실시간 서비스 모니터링: 서비스 운영 현황을 실시간으로 시각화 하여 문제 발생 시 즉각 대응 지원 AI 기반 이미지 인식: 이미지 자동 분석을 통해 표준 준수 여부 확인 및 개선 필요 항목 식별 디지털 체크리스트 및 할 일 관리: 업무 할당을 체계화하고 적시 수행 여부를 관리 TPO 기반 표준 설정: 시간(Time), 장소(Place), 상황(Occasion)에 따른 유연한 관리 표준 설정 자동 보고서 생성: 서비스 성과 보고서를 자동 생성하여 트렌드 분석 및 개선점 도출 사용자 맞춤형 대시보드: 핵심 지표(KPI)를 한눈에 파악할 수 있는 개인화된 데이터 화면 제공 시스템 연동: 기존 운영 시스템과의 데이터 통합으로 업무 연속성 보장 VOC(고객의 소리) 분석: 고객 피드백 분석을 통한 문제 해결 및 서비스 고도화 실시간 알림: 이슈 발생 시 즉각적인 알림 발송으로 관리자 대응 속도 향상 |

## 슬라이드 3

이미 시장 검증이 끝난 모니터링(단순 관제)을 넘어, 현장의 실질적인 변화를 이끌어내는

'Action 시스템'에 집중하여 MVP(Minimum Viable Product) 개발을 위한

PRD(제품 요구사항 정의서)와 기능요구정의서를 작성해 드립니다.

MVP의 핵심은 데이터가 어떻게 액션으로 이어지고, 그 액션이 어떻게 다시 수치화(KPI)되는가"입니다.


| MVP 개요 | 제품명 | CS MAC (Service Operations OS - Action Module) |
| --- | --- | --- |
|  | 목표 | 담당자만 알고 있는 정보가 회사가 알고 있는 정보로 전환되어 리스크관리 비용 절감을 증명한다 |
|  | 목적 | TPO 기준에 따른 현장 실행력을 확보하고, AI 판독을 통해 실시간 실행 근거를 검증하며 객실관리, 매장관리, 시설관리 기반의 운영 효율성 KPI를 도출함 |
|  | 핵심가치 | "Check(감시)"가 아닌 "Action(수행)"과 "Analysis(최적화)"의 연결 |
| 핵심 타겟 페르소나 | Actioner (현장 실무자) | 복잡한 판단 없이 지시된 TPO 카드를 수행하고 근거만 남김 |
|  | Controller (현장 관리자) | 보직·경험과 무관하게 ‘어떻게 지시해야 하는지’를 알려주는 표준 업무지시 라이브러리”(*해당 직무 전문성이 없는 관리자들의 역량 편차를 줄여주는 용도) |
|  |  | AI가 필터링한 실행 근거를 확인하고, 병목 구간(리드타임)을 관리함 |
| 핵심 유저 스토리 | 관리자는 시간/장소/상황에 맞는 서비스 표준(TPO)을 설정하고 이행근거를 요청 할 수 있다 |  |
|  | 실무자는 본인에게 할당된 '직무카드'를 확인하고 시작/종료를 통해 리드타임과 이행근거를 생성한다 |  |
|  | 실무자가 촬영한 인증샷은 AI가 판독하여 관리자 대시보드에 우선순위별로 나열된다 |  |
|  | 시스템은 수집된 실행 데이터를 바탕으로 준수율 및 리드타임 KPI를 자동 생성한다 |  |
